GitHub Actionsでブログをデプロイする

March 7, 2021 · 7 min read

sugu-sano

web application developper

前書き#

GitHub の静的ファイルホスティングサービス、GitHub Pages にブログをデプロイするために設定を行い、実際にブログをデプロイしました。

Docusaurus にはもともと簡単に GitHub Actions でデプロイできるように設定されています。

そのため少しのカスタマイズで GitHub Pages にブログをホスティングすることが可能です。

デプロイ準備#

GitHub にリポジトリを作成#

新しいリポジトリの作成 | GitHub Docs

GitHub にブログのリポジトリを作成し push しておきます。

そして GitHub Actions に使用するシークレットをリポジトリにを登録します。

⚙️Setting > Secrets > New repository secret

キー名はGH_PAGES_DEPLOYで値は~/.ssh/ディレクトリにある GitHub への接続する SSH 秘密鍵の値です。

サイトの設定#

docusaurus.config.jsを編集するとサイトの基本的な設定を行うことができます。今回は下記のように設定しました。

module.exports = {
title: "nossa dev",
tagline: "The tagline of my site",
url: "https://sugu-sano.github.io",
baseUrl: "/dev-blog/",
projectName: "dev-blog",
organizationName: "sugu-sano",
//...
};

urlはブログの URL です。GitHub Actions にデプロイされるため、カスタムドメインを設定しなければhttps://[ユーザ名（組織名）].github.ioを設定します。

baseUrlはブログのルートパスの設定で、この場合、トップページがhttps://sugu-sano.github.io/dev-blog/になります。

projectNameはリポジトリ名そのままで、organizationNameは個人の場合は GitHub のユーザ名になります。

ブログのナビゲーションバーのリンクを修正#

今回はブログだけ欲しいのでthemeConfig.navbar.itemsを下記のように編集しドキュメントへのリンクを消しました。 それと、Github へのリンクを自分のブログのリポジトリにしておきます。

items: [
{ to: "blog", label: "Blog", position: "left" },
{
href: "https://github.com/sugu-sano/dev-blog",
label: "GitHub📦",
position: "right",
},
],

ブログのフッターの修正#

フッターにあるDocusaurusのへのリンクも消してます。 themeConfig.footer.linksを空配列にしておきます。

links: [],

フッターのコピーライトの修正#

このブログではアイコンに flatticon を使用しているのでフッターにリンクを貼ります。 themeConfig.footer.copyrightの文字列は HTML を書いても解釈して表示してくれるようです。

copyright: `
<div>Icons made by <a href="https://www.freepik.com" title="Freepik">Freepik</a> from <a href="https://www.flaticcom/" title="Flaticon">www.flaticon.com</a></div>
Copyright © ${new Date().getFullYear()} sugu-sano. Built with Docusaurus.`,

Github Actions のワークフロー用 YAML ファイルを作成#

.github/workflowsディレクトリを作成しdocumentation.ymlファイルを作成します。

下記を参考にしました。

3.2.2 .github/workflows/documentation.yml ファイルの作成 | ドキュメント作成ツールの決定版！Markdown + React の体験を Docusaurus で

ここから自分にあったかたちに変更しました。

もともと、documentationブランチをプッシュをトリガーにデプロイされるようになっていましたが、デプロイ事故を防ぎたいので GitHub の Actions タブで任意にデプロイされるようにしました。

on:
workflow_dispatch: # Action タブから任意に実行できるようにする

checksとgh-releaseジョブで使用される Node.js のバージョンを volta で固定したバージョンと同じにしました。

- uses: actions/setup-node@v1
with:
node-version: "14.15.5"

環境変数GIT_USERにトークンも渡すようにします。

env:
USE_SSH: true
GIT_USER: ${{ github.actor }}:${{ github.token }}
最終的な YAM Lファイル
name: deploy
on:
workflow_dispatch: # Action タブから任意に実行できるようにする
jobs:
checks:
if: github.event_name != 'push'
runs-on: ubuntu-latest
steps:
- uses: actions/checkout@v1
- uses: actions/setup-node@v1
with:
node-version: "14.15.5"
- name: Test Build
run: |
if [ -e yarn.lock ]; then
yarn install --frozen-lockfile
elif [ -e package-lock.json ]; then
npm ci
else
npm i
fi
npm run build
gh-release:
if: github.event_name != 'pull_request'
runs-on: ubuntu-latest
steps:
- uses: actions/checkout@v1
- uses: actions/setup-node@v1
with:
node-version: "14.15.5"
- name: Add key to allow access to repository
env:
SSH_AUTH_SOCK: /tmp/ssh_agent.sock
run: |
mkdir -p ~/.ssh
ssh-keyscan github.com >> ~/.ssh/known_hosts
echo "${{ secrets.GH_PAGES_DEPLOY }}" > ~/.ssh/id_rsa
chmod 600 ~/.ssh/id_rsa
cat <<EOT >> ~/.ssh/config
Host github.com
HostName github.com
IdentityFile ~/.ssh/id_rsa
EOT
- name: Release to GitHub Pages
env:
USE_SSH: true
GIT_USER: ${{ github.actor }}:${{ github.token }}
run: |
git config --global user.email "actions@gihub.com"
git config --global user.name "gh-actions"
if [ -e yarn.lock ]; then
yarn install --frozen-lockfile
elif [ -e package-lock.json ]; then
npm ci
else
npm i
fi
npx docusaurus deploy

サイドバーの修正#

サイドバーのドキュメントへのリンクもブログには不要なので消してしまいます。

module.exports = {
someSidebar: {
Docusaurus: [],
Features: [],
},
};

トップページの修正#

もともと Docusaurus の説明などいろいろコンテンツがありましたが、すべて不要なので削除します。

トップページ/dev-blogにアクセスされた場合、メインコンテンツのブログ/dev-blog/blogヘリダイレクトされるようにします。

リダイレクト処理は"@docusaurus/router"のRedirectコンポーネントで簡単に可能でした。

import React from "react";
import { Redirect } from "@docusaurus/router";
function Home() {
return <Redirect to="/dev-blog/blog" />;
}
export default Home;

サンプルのマークダウンファイルを削除#

最後に/docsと/blogsディレクトリ以下にあるマークダウンファイルを削除します。

mdx.mdだけは消すとビルドに失敗するので消すことができませんでした。

（依存関係をご存知の方がいらっしゃったら教えてください）

実際にデプロイする#

すべての変更を push したら GitHub にアクセスし Actions タブを開きます。

そして、Run workflowを選択しmainブランチを洗濯して実行します。

1 分弱待つとhttps://sugu-sano.github.io/dev-blog/にデプロイされます。
